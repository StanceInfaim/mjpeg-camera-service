# MJPEG Camera Service

[![Build APK](https://github.com/StanceInfaim/mjpeg-camera-service/actions/workflows/build.yml/badge.svg)](https://github.com/StanceInfaim/mjpeg-camera-service/actions/workflows/build.yml)
[![Release](https://img.shields.io/github/v/release/StanceInfaim/mjpeg-camera-service?sort=semver)](https://github.com/StanceInfaim/mjpeg-camera-service/releases/latest)

**[English](README.md) | Русский**

Безголовое Android-приложение (без Activity, без UI), превращающее телефон в MJPEG IP-камеру по обычному HTTP. Изначально сделано, чтобы дать 3D-принтеру на Klipper/KlipperScreen дешёвую и надёжную веб-камеру из старого смартфона, но подходит как универсальная сетевая камера для всего, что умеет читать MJPEG-поток.

Всё приложение — это один foreground-`Service`: он снимает кадры через Camera2 и отдаёт их встроенным мини-HTTP-сервером. Без внешних библиотек.

## Возможности

- **MJPEG-поток** и **одиночный JPEG-снимок** по HTTP.
- **Веб-страница настроек** `/settings` — всё настраивается из браузера, без пересборки.
- **Настраиваются** камера (фронтальная/основная), разрешение, FPS, качество JPEG, поворот, горизонтальное отражение, компенсация экспозиции, режим автофокуса и HTTP-порт. Настройки переживают перезагрузку.
- **Гибридный захват** — используется аппаратный JPEG-энкодер камеры, когда выбранное разрешение его поддерживает (почти нулевая нагрузка на CPU), и прозрачный откат на программное сжатие (`YUV_420_888` → JPEG) для размеров, которых нет в аппаратном списке.
- **Эффективная отдача** — клиенту уходят только реально новые кадры (без дубликатов, без активного опроса), с ограничением до заданного FPS.
- **Отказоустойчивость** — самоперезапуск при краше (AlarmManager + `START_STICKY`), переоткрытие камеры при отключении, удержание wake lock, чтобы поток жил при выключенном экране.

## Требования

- Android-телефон, **Android 5.0 (API 21) или новее**.
- **Для штатного использования нужен root** (автономный запуск при загрузке). См. [Зачем нужен root](#зачем-нужен-root).
- Настроенный `adb` на компьютере для первой установки.

## Скачать

Последний APK — на странице [**Releases**](https://github.com/StanceInfaim/mjpeg-camera-service/releases/latest), либо соберите сами (см. [Сборка](#сборка-из-исходников)).

## Быстрый старт

```bash
# 1. Установка
adb install mjpeg-camera-service-v2.0.0.apk

# 2. Выдать разрешение на камеру (UI для запроса нет)
adb shell pm grant com.stanceinfaim.mjpegcamera android.permission.CAMERA

# 3. Запустить сервис
adb shell am start-foreground-service com.stanceinfaim.mjpegcamera/.CameraStreamService
```

Затем откройте в браузере `http://<ip-телефона>:8080/stream`, либо `http://<ip-телефона>:8080/settings` для настройки.

> IP телефона можно узнать командой `adb shell ip -o addr show wlan0`.

## Веб-интерфейс и HTTP API

Порт по умолчанию — **8080**.

| Путь | Метод | Поведение |
|---|---|---|
| `/settings` | GET | Веб-страница настроек |
| `/api/settings` | GET | Текущие настройки + поддерживаемые разрешения (JSON) |
| `/api/settings` | POST | Применить настройки (JSON в теле); сохраняется в хранилище |
| `/`, `/video`, `/stream` | GET | MJPEG-поток (`multipart/x-mixed-replace`) |
| `/shot`, `/capture` | GET | Одиночный JPEG-снимок |
| `/front` | GET | Переключиться на фронтальную камеру |
| `/back` | GET | Переключиться на основную камеру |
| `/status` | GET | Текущая камера, порт, разрешение, FPS (JSON) |

### Настройки

| Настройка | Диапазон / значения | Примечание |
|---|---|---|
| Камера | основная / фронтальная | id Camera2 `"0"` / `"1"` |
| Разрешение | из списка устройства | Размеры с меткой `(SW)` идут через программное сжатие |
| FPS | 1–60 | Верхняя граница, до которой ограничивается поток |
| Качество JPEG | 10–95 | Главный рычаг управления трафиком |
| Поворот | 0 / 90 / 180 / 270° | Аппаратный на JPEG-пути |
| Горизонтальное отражение | вкл / выкл | Программное (декод → отражение → энкод) |
| Компенсация экспозиции | диапазон устройства | Полезно для тёмных/ярких сцен |
| Автофокус | непрерывный / фиксированный | |
| HTTP-порт | 1024–65535 | Смена перезапускает сервер |

## Интеграция с Klipper

В **Mainsail** или **Fluidd** добавьте веб-камеру:

- **URL потока:** `http://<ip-телефона>:8080/stream`
- **URL снимка:** `http://<ip-телефона>:8080/shot`
- **Сервис / тип:** `MJPEG-Streamer (adaptive)` или любой вариант «MJPEG stream».

Те же URL работают в **KlipperScreen**, OctoPrint, Frigate, Home Assistant и любом другом ПО, понимающем MJPEG/HTTP-поток.

## Автозапуск при загрузке

Приложение **не** запускает себя само при загрузке — и это сделано намеренно (см. ниже). Запускайте его при загрузке из привилегированного (root) контекста. С Magisk проще всего использовать скрипт Termux:Boot или скрипт в `service.d`, выполняющий:

```sh
su -c "am startforegroundservice -n com.stanceinfaim.mjpegcamera/.CameraStreamService"
```

### Зачем нужен root

Установить и запустить приложение вручную через `adb` можно и без root. Root нужен для реального сценария — **автономного запуска при загрузке с доступом к камере**:

- На **Android 11+** foreground-сервис, запущенный из фонового broadcast (например, внутренним приёмником `BOOT_COMPLETED`), не получает доступ к камере из-за политики разрешений *while-in-use* — камера сообщает `CAMERA_DISABLED ... disabled by policy`. Привилегированный `am startforegroundservice` (root/shell) получает временное исключение, и камера открывается. Поэтому приложение полагается на root-скрипт запуска, а не на собственный boot-приёмник.
- Выдача разрешения на камеру приложению без UI и его запуск при загрузке реально возможны только из привилегированного контекста.

## Нюансы

- **Нет аутентификации и TLS.** Поток открыт всем в сети. Держите телефон в доверенной/изолированной локальной сети.
- **Ограничивает FPS обычно сеть, а не приложение.** На высоком разрешении и качестве один JPEG может весить сотни КБ; на слабом Wi-Fi это и упирает частоту кадров. Снижайте сначала разрешение или качество JPEG.
- В слабом освещении автоэкспозиция камеры снижает реальную частоту кадров ради яркости картинки, поэтому отдаваемый FPS может быть ниже заданного предела.
- id камер `"0"`/`"1"` — это соглашение Camera2; на некоторых устройствах физическое соответствие отличается.

## Сборка из исходников

Для тулчейна сборки нужен JDK 17 (совместимость исходников/таргета — Java 8).

```bash
./gradlew assembleDebug     # app/build/outputs/apk/debug/app-debug.apk
./gradlew assembleRelease   # app/build/outputs/apk/release/app-release.apk
```

## Релизы и CI

- **CI** ([`build.yml`](.github/workflows/build.yml)) собирает debug-APK на каждый push и pull request в `main`.
- **Релизы** ([`release.yml`](.github/workflows/release.yml)) собирают APK и прикрепляют его к GitHub Release при пуше тега `v*`:

  ```bash
  git tag v2.0.0
  git push origin v2.0.0
  ```

  Чтобы выпускать корректно подписанный релиз, добавьте секреты репозитория (иначе сборка использует debug-ключ — такой APK тоже устанавливается):

  | Секрет | Описание |
  |---|---|
  | `KEYSTORE_BASE64` | base64 вашего `.jks`-хранилища |
  | `KEYSTORE_PASSWORD` | пароль хранилища |
  | `KEY_ALIAS` | алиас ключа |
  | `KEY_PASSWORD` | пароль ключа |
